Системные вызовы, используемые tinyhttpd:

* sendto - отправляйет байты в сокет;
* read - читает байты из файла;
* recvfrom - получает байты из сокета;
* close - закрывает файловый дескриптор;
* mmap - "отображает файл на память" (чтение и запись файла с буферизацией);
* write - записывает байты в файл;
* clone - аналог fork (создание нового процесса), но с общим
  доступом к адресному пространству (для организации многопоточности, есть только в Linux)
* accept - ожидает поступление запроса на соединение
  (или извлекает из очереди запросов на соединение первый);
* mprotect - изменяет права доступа к участку памяти (в которую уже отображён файл);
* open - открывает доступ на чтение/запись файла
* wait4 - ожидает завершения работы другого процесса (pid которого известен)
* socket - создаёт сокета;
* bind - привязывает сокет к IP-адресу (и порту)
* stat - информация о файле по его имени
  (размер, владелец, права доступа, время создания и т.п.)
* pipe - создание однонаправленного неименованного канала для
  передачи информации между родственными процессами
* fstat - аналог stat, но для открытого файла (отличается передаваемым аргументом функции)
* munmap - удаляет отображение файла на память (которое сделалось вызово mmap)
* access - проверяет, имеет ли вызывающий процесс права доступа к файлу
* listen - переводит сокет в режим ожидания соединений
* rt_sigaction - установление функции-обработчика
  сигнала, переданного процессу (например, сигнала SIGTERM)
* getrlimit - получает установленные лимиты на использование
  системных ресурсов вызывающим процессом
* arch_prctl - какая-то линуксовая фигня для 64-битных процессоров
* set_tid_address - что-то совсем непонятное
* set_robust_list - что-то с мьютексами (аналогами семафоров) связано
* rt_sigprocmask - проверяет и/или устанавливает сигналы, которые процесс будет получать
  (используется чтобы не принимать, например, сигналы завершения процесса, когда выполняется
   критически важная часть программы)
* brk - выделяет или освобождает память для процесса
* execve - запуск программы (бинарника или скрипта)
